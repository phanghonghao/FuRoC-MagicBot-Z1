1 策略梯度直觉

策略 π 是什么？

策略是一个函数：给定观测 → 输出动作的概率分布。对 Z1 来说，输入 235 维观测，输出 12 个关节各自的高斯分布（均值 + 标准差）。

为了直观理解，下面把动作简化为 1 维（比如"左膝弯曲程度"），策略就是一个高斯分布曲线。点击按钮采样，看分布如何更新。

绿色区域 = 高奖励。采样后，策略（蓝色曲线）会朝奖励更高的方向移动。这就是策略梯度的核心思想。

直觉总结

策略梯度一句话：采样动作 → 看奖励 → 增大好动作的概率，减小坏动作的概率。

问题是：如果学习率太大，一步更新太猛，策略可能崩溃。这就是 PPO 要解决的核心问题。

2 PPO Clip — 核心创新

问题：更新太大会崩溃

如果策略一步变太多（比如突然把所有概率都翻转了），机器人行为会突变，之前学到的全部白费。这叫策略崩溃。

解法：限制变化幅度

PPO 引入了一个概率比：

r(θ) = π新(a|s) / π旧(a|s)

r = 1.0 表示新旧策略一样。r = 2.0 表示新策略认为这个动作的概率是旧的 2 倍。

PPO 的目标函数：

LCLIP = E[ min( r·A , clip(r, 1−ε, 1+ε)·A ) ]

其中 A 是优势函数（这个动作比平均水平好多少），ε 是 clip 范围（Z1 用 0.2）。

互动：拖动滑块理解 Clip

下面的图展示了 对于单个 (r, A) 组合，目标函数的形状。调整参数观察：

r·A 无裁剪 clip(r)·A 裁剪后 PPO 实际目标 min(·,·) 裁剪激活区
A > 0（好动作）：clip 防止 r 变得太大，即防止"好动作概率暴增"。 | 灰色虚线标记 r=1（新旧策略相同）。
关键理解：当 r 在 [1-ε, 1+ε] 范围内时，PPO 和普通策略梯度一样。一旦超出，目标函数被"削平"，梯度为零，策略不会再往那个方向更新。这就是"近端 (Proximal)"的含义——新策略不能离旧策略太远。

3 GAE — 优势估计

什么是"优势"？

优势 A(s,a) 回答这个问题："这个动作比平均水平好多少？"

  • A > 0 → 比平均好，PPO 会增大它的概率
  • A < 0 → 比平均差，PPO 会减小它的概率
  • A = 0 → 平平无奇，不更新

怎么算优势？两难选择

最直觉的两种极端：

• TD(0)：只用"当前奖励 + 下一状态的价值估计"。方差低，但偏差大（依赖价值函数准不准）。

• Monte Carlo：等整个 episode 跑完，累加所有未来奖励。偏差低，但方差大（一条轨迹可能很 noisy）。

GAE (Generalized Advantage Estimation) 用参数 λ 在两者之间插值：

Ât = δt + (γλ)δt+1 + (γλ)2δt+2 + ...
其中 δt = rt + γV(st+1) − V(st) 是 TD 残差

互动：调节 λ 看效果

下面模拟了一条 10 步轨迹的奖励。拖动 λ 滑块，观察每步的优势估计如何变化。

上：轨迹奖励 (蓝色柱) 和价值函数估计 (灰色虚线)。下：各步的优势估计。λ 越大，优势"传播"越远（低偏差高方差）；λ 越小，只看近处（高偏差低方差）。
Z1 的配置：γ=0.99, λ=0.95。这是 RL locomotion 的经典组合——接近 MC 但略有平滑，适合连续控制任务。

把前三节串起来，就是一次完整的 PPO 更新。点击"下一步"逐步看。

1. 环境交互 — 采集数据

↓
2. 计算回报 & 优势
GAE(λ=0.95) 计算每步优势 Ât，折扣回报 Rt
↓
3. 计算旧策略概率
记录 log πold(a|s)，后续 ratio r = πnew/πold
↓
4. PPO 更新 (5 epochs × 4 mini-batches)
每个 mini-batch 24,576 条数据，计算 clip loss + value loss + entropy
↓
5. 梯度下降，更新 Actor + Critic
lr=1e-3, max_grad_norm=1.0 → 策略微微变好
↓
6. 回到第 1 步，用新策略采集数据
重复 50,000 次迭代
点击"下一步"开始。

# RTX Pro 6000D 服务器使用守则 (pro6000d)

**服务器地址**: `192.168.120.155`
**硬件**: 256核 Intel Xeon 6767P / 1.5TB RAM / **8× NVIDIA RTX 6000D (85GB)**
**排队系统**: Slurm 21.08.5 (单节点模式)

---

## 1. GPU 资源申请（核心规则）

1. **所有 GPU 训练任务必须通过 Slurm 提交**，禁止直接 `CUDA_VISIBLE_DEVICES=x python train.py`。
2. 短时调试（<30 分钟、1 张卡）可容忍直接运行，但确认无误后应改为 Slurm 提交。
3. 多卡任务（2 卡及以上）**必须**使用 Slurm，避免冲突。
4. 长任务、批量任务、定时任务均走 Slurm。

## 2. Slurm 使用方法

### 2.1 交互式运行 (srun)

```bash
# 申请 1 张 GPU，8 个 CPU 核心，最多运行 4 小时
srun --gpus=1 -c 8 -t 4:00:00 python train.py

# 申请 4 张 GPU，16 核，8 小时
srun --gpus=4 -c 16 -t 8:00:00 torchrun --nproc_per_node=4 train.py

# 申请 bash 终端（调试用）
srun --gpus=1 -c 4 -t 1:00:00 --pty bash
```

### 2.2 批处理提交 (sbatch)

```bash
#!/bin/bash
#SBATCH -J p3_fine                # 任务名
#SBATCH -p gpu                    # 分区 (gpu=默认, cpu=无GPU)
#SBATCH --gpus=1                  # GPU 数量
#SBATCH -c 8                      # CPU 核数
#SBATCH -t 24:00:00               # 最长运行时间
#SBATCH -o logs/%x-%j.out         # 输出日志
#SBATCH -e logs/%x-%j.err         # 错误日志

source ~/.bashrc
conda activate <your_env>

python train.py --config configs/train.yaml
```

提交: `sbatch job.slurm`

### 2.3 常用命令

| 命令 | 用途 |
|------|------|
| `sinfo` | 查看节点/分区状态 |
| `squeue -u $USER` | 查看自己的作业 |
| `squeue` | 查看所有作业 |
| `scontrol show job <id>` | 查看作业详情 |
| `scancel <jobid>` | 取消作业 |
| `watch_gpu` | 查看 GPU 占用 + 是否经 Slurm 分配 |

### 2.4 分区说明

| 分区 | GPU | 用途 | 默认时间 |
|------|-----|------|----------|
| `gpu` (默认) | 有 (最多 8 卡) | 训练任务 | 1 小时 |
| `cpu` | 无 | CPU 计算、数据处理 | 无限制 |

### 2.5 使用建议

1. 先用小数据、小步数做短时验证，再提交长任务。
2. 日志目录请按项目和日期区分，便于排障。
3. 不要反复提交无效任务占用排队资源。
4. 估算好时间，设置合理的 `-t` 限时，避免资源浪费。

## 3. 带宽使用与报备

1. 长时间占用带宽（大规模下载/上传、数据同步、镜像拉取）需在群内报备。
2. 报备信息包含：开始时间、预计时长、数据规模、用途。
3. 高峰时段主动限速或错峰传输。

## 4. 严禁行为

1. 严禁挖矿、养虾等与科研无关的资源牟利行为。
2. 严禁任何违法行为（入侵、DDoS、钓鱼、赌博、传播违法内容等）。
3. 严禁违反学校、学院、实验室管理规定。
4. **严禁绕过 Slurm 直接多卡占用**，一经发现将通知整改。

## 5. 账号与数据安全

1. 账号仅限本人使用，禁止共享或转借。
2. 妥善保管密码与令牌，不得明文存放在公共位置。
3. 重要数据定期备份，删除文件前确认避免误删。

## 6. 违规处理

1. 口头警告；
2. 限制或暂停账号权限；
3. 涉嫌违法的，移交相关部门处理。

## 7. 存储使用规范

1. 数据放数据盘，系统盘仅保留运行环境与脚本。
2. 单用户数据盘占用不超过总容量 50%。
